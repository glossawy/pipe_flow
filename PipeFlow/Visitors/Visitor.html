#visit(object) ⇒ Object

Note:

It is recommended that visitor implementations extend DSL rather than write method names manually.

Start a visitation recurrence for the given AST. It is expected that for any relevant visitable classes of objects there exist a method for that class. Given a class with the name Some::Module::Path, there ought to be a visit_Some_Module_Path method taking the visited object as an argument.

If a method is not defined for the object's class, then if any methods are defined for visiting any of an object's superclasses, visitation will be attempted on that method.

Examples:

Visiting an Object by Class

class SomeVisitor < Visitor
  def visit_String(str)
    # ...
  end
end
SomeVisitor.new.visit('abc')

Visiting an Object by Superclass

class SomeVisitor < Visitor
  def visit_Object(obj)
    # ...
  end
end
SomeVisitor.new.visit('abc')

    Any object, though typically an AST::Base instance

def visit(object)
  find_visitable_method_for(object.class) do |visit_method_id|
    return send(visit_method_id, object)
  end

  raise TypeError, "Unable to visit #{object.class}"
end
